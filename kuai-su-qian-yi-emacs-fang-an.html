快速迁移 emacs 方案

简便迁移 emacs

目的

当需要在新的电脑上使用 emacs 的时候, 快速轻便的搭建自己的 emacs 环境. 要实现的直接目标即: 需要迁移自己的 emacs 环境的时候, 一个命令即可.

实现方法

emacs 本身是由插件支持的, 通过配置设置和配置插件来达到个性化的目的. 因此关键的目录只有两个 .emacs 配置文件和 .emacs.d 插件目录.

配置的迁移比较简单 直接上传 gist 服务器上, 从 gist 服务器下载即可. 国内因为 gist 服务被屏蔽了, 虽然可以通过修改 dns 正常访问, 为了省却这个改 dns 的过程, 直接拿 ip 访问了.

插件迁移因为插件数目多, 下载文件也多, 略麻烦. 有两种思路, 第一种是插件也完全上传到 github 服务器. 但这样做不够轻便. 另外一种思路是通过 emacs 内置提供的 package 功能来实现配置自动下载. 这种方案只需要将需要安装下载的插件 写进配置即可.

具体操作

准备 emacs 程序

在需要迁移的操作系统环境上, 使用操作系统提供的下载工具下载最新的 emacs(当前是 emacs24). centos 使用 yum install emacs

ubuntu 使用命令 sudo apt-get install emacs osx 使用 brew install emacs

执行迁移

打开 emacs 程序 在 buffer 中粘贴下面的代码

(defun curl-save (url file)
  (interactive)
   (with-current-buffer (url-retrieve-synchronously  url)
  (goto-char (point-min))
  (re-search-forward "\r?\n\r?\n" nil t)
  (delete-region (point) (point-min))
  (write-region (point-min) (point-max) file))
)
(mkdir "~/.emacs.d")
(curl-save "https://103.245.222.133/whispermemory/48d735397ce955c4b9b8/raw/89855d08ff82e2ac6f0c4ac2970224c30d967c67/.emacs" "~/.emacs")
(curl-save "https://103.245.222.133/whispermemory/48d735397ce955c4b9b8/raw/89855d08ff82e2ac6f0c4ac2970224c30d967c67/auto-config.el" "~/.emacs.d/auto-config.el")
(curl-save "https://103.245.222.133/whispermemory/48d735397ce955c4b9b8/raw/89855d08ff82e2ac6f0c4ac2970224c30d967c67/auto-package.el" "~/.emacs.d/auto-package.el")

重启 emacs 之后会自动更新配置并安装插件

代码解释

这段代码主要使用 url-retrieve-synchronously 函数进行下载, 之后进行一些文件操作 详见 url package

curl-save 执行的是下载并安装的函数

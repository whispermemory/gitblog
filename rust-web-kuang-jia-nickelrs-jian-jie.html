rust web 框架 nickel.rs 简介

介绍

nickel.rs 是一个使用 rust-lang 语言实现的简单轻量级的 web 框架. 风格类似 nodejs 的 express 框架. 比较简洁. 本文介绍使用 nickel.rs 搭建 web 服务的流程.

初步认识 nickel.rs

简单的看下运行一个 hello, world 的 http 服务代码

#[macro_use] extern crate nickel;

use nickel::{Nickel, HttpRouter};

fn main() {
    let mut server = Nickel::new();
    server.get("**", middleware!("Hello World"));
    server.listen("127.0.0.1:6767");
}

代码简单明了!

准备工作

  • 使用 binary 文件安装 rust-lang v1.0.0 (手动编译时间较长, 使用 binary 安装较为便捷)
  • 安装 cargo , cargo 是 rust-lang 的包管理工具, 如果使用 rust 的 binary 安装,则会自带 cargo, 可省略这一步.
  • checkout nickel.rs 代码, github 地址 https://github.com/nickel-org/nickel.rs.git

灵活的 routing

#[macro_use] extern crate nickel;

use nickel::{Nickel, HttpRouter};

fn main() {
    let mut server = Nickel::new();

    server.get("/bar", middleware!("This is the /bar handler"));
    server.get("/user/:userid", middleware! { |request|
        format!("This is user: {:?}", request.param("userid"))
    });
    server.get("/a/*/d", middleware!("matches /a/b/d but not /a/b/c/d"));
    server.get("/a/**/d", middleware!("This matches /a/b/d and also /a/b/c/d"));

    server.listen("127.0.0.1:6767");
}

router 支持下面几种格式

  • 单一的 router 变量: /router/:router_argument

  • 通配 router : /some_matched/*/route

  • 双通配符 router: /a/**/route , 可以通配 /a/b/route a/b/c/route

router 和 request 搭配使用

先看看 get 方法:

server.get("/", middleware!("This is the /bar handler"));
//获取 request 参数
server.get("/user/:userid", middleware! { |request|
        format!("This is user: {:?}", request.param("userid"))
    });

//返回数据
 server.get("/", middleware! { |_, response|
        let mut data = HashMap::new();
        data.insert("name", "user");
        return response.render("examples/assets/template.tpl", &data);
    });

post 方法

server.post("/a/post/request", middleware! { |request, response|
        format!("Hello {}","post hello")//注意此处无 ;
    });

json 处理, 通常会转化为结构体

#[derive(RustcDecodable, RustcEncodable)]
struct Person {
    firstname: String,
    lastname:  String,
}
server.post("/a/post/request", middleware! { |request, response|
        let person = request.json_as::<Person>().unwrap();
        format!("Hello {} {}", person.firstname, person.lastname)
});

使用模板

nickel.rs 使用 mustache 的模板 , 可以参照 demo , demo 中使用的是 json 传递后端口.

<html>
    <body>
        <h1>
            Hello {{ name }}!
        </h1>
    </body>
</html>

在后端, 使用 response.render(template, data) 将数据渲染到 html 中

#[macro_use] extern crate nickel;
use std::collections::HashMap;
use nickel::{Nickel, HttpRouter};

fn main() {
    let mut server = Nickel::new();

    server.get("/", middleware! { |_, response|
        let mut data = HashMap::new();
        data.insert("name", "user");
        return response.render("examples/assets/template.tpl", &data);
    });

    server.listen("127.0.0.1:6767");
}

如果需要传递数组, 并解析数组 template 可以写成

{{#names}} {{.}} {{/names}}

后端代码如下

 #[macro_use] extern crate nickel;
 use nickel::{Nickel, HttpRouter};
 use std::collections::HashMap;

 fn main() {
     let mut server = Nickel::new();
     server.get("/", middleware! { |_, response|
                                       let mut data = HashMap::new();
                                       let v = vec!["zhang","wang","li"];
                                       data.insert("items", v);

                                       return response.render("examples/assets/template.tpl", &data);
     });

     server.listen("127.0.0.1:6767");
 }

另外在 mustache 中 # 也可以代表存在 true, 如果是不存在或 false 使用 ^ {{ ^exists }} {{ do something }} {{/exists}}

一些不太方便的地方

  • 因为使用了宏, 使用相对来说比较简洁方便. 宏提升了学习曲线.
  • 不支持热编译, 改写后端代码后, 需要重新编译运行
  • 前端模板变化, 需要重新编译
